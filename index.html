Registack AIR

Agent Detector CLI

Local-only detector for AI-agent candidates, runtime artifacts, Docker workloads, and Kubernetes manifests. Install from the terminal, select a persistent default path by number, and emit AIR-compatible JSON with persistent new-installation detection inside that selected path. The detector is a free client. Central registry intake and agent registration stay in the licensed Registack AIR Control Plane.

macOS

Install and choose the persistent default detection path by number or through Finder. If you want full-machine coverage, choose filesystem root /.

curl --http1.1 -fsSL https://www.registack.eu/cli/registack-agent-detector/install-macos.sh | bash

Linux

Install and choose the persistent default detection path by number or desktop file picker when supported. If you want full-machine coverage, choose filesystem root /.

curl --http1.1 -fsSL https://www.registack.eu/cli/registack-agent-detector/install-linux.sh | bash

Windows

Run the PowerShell installer and select the persistent default detection path by number or through File Explorer.

powershell -ExecutionPolicy Bypass -c "irm https://www.registack.eu/cli/registack-agent-detector/install-windows.ps1 | iex"

Verify

Once installed, verify the CLI, run the saved persistent default path explicitly, and inspect the bundled AIR importer.

registack-agent-detector --version
registack-agent-detector --scan-default --output json
registack-agent-detector --scan-default --review
registack-air-import --version
registack-air-link --version

Tenant Binding

Register or bind a tenant in the AIR Control Plane, then attach the local detector to that tenant and optionally register this machine as a managed device.

registack-air-link --base-url http://127.0.0.1:8092/admin/installer --token "$REGISTACK_AIR_TOKEN" register-tenant --customer-id customer_000001 --name "Pilot Tenant" --slug pilot-tenant --bind
registack-air-link --base-url http://127.0.0.1:8092/admin/installer --token "$REGISTACK_AIR_TOKEN" register-device --device-label "Eduard MacBook Air"

AIR Import

The detector JSON now contains an air_candidate block per detection, plus a guideline-based high_risk_assessment and an AIR-shaped applicability_hint. These labels are derived from the European Commission draft guidelines on the classification of high-risk AI systems published on 19 May 2026. Use the bundled importer to review detections locally, record them as reviewed, and then explicitly decide whether to proceed to AIR detection intake. The review output now also surfaces the high-risk and applicability snapshot, the parsed actual_agent_profile, resolved profile sources, and captured skill/context/prompt artifact previews. After tenant binding, the importer can reuse the saved AIR base URL and tenant ID automatically.

registack-agent-detector --scan-default --output json | registack-air-import --review --dry-run

Export JSON File

Write the detector result directly to a JSON file when you want to hand the output into AIR Control Plane, archive a scan, or send it to another review step.

registack-agent-detector --scan-default --json-file ./detected-agents.json
registack-agent-detector --scan-default --json-file
registack-agent-detector --scan-default --output json --output-file ./detected-agents.json
registack-agent-detector --scan-default --output json --output-file

If you omit the file path, the detector opens the native save dialog so the target location can be selected without manual path typing.

Actual Agent Profile

For managed or otherwise profile-bearing agents, the detector JSON also includes an agent_identity block. It can expose the detected agent root, identity bundle paths, the parsed actual-profile.json, and preview content from skill and context artifacts such as SKILL.md, README.md, manifest.json, and prompts.json.

Operator Notes

No manual path typing is required during installation. The installer presents a numbered list of valid detection paths and can also open a system folder picker. It stores the selected path persistently until uninstallation and marks newly observed agent installations inside that path as new on later scans. Reviewed detections are then kept locally until they are transferred into the central AIR control plane.
